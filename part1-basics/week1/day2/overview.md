# Week 1 - Day 2: AI Ethics, Security & Responsible Use

## Learning Objectives

Understanding ethical implications, security risks, and responsible practices when using AI tools for software development.

## Success Criteria

By the end of Day 2, you should be able to:

1. **List 5 security risks when using AI coding tools**
   - Identify data leakage and privacy concerns
   - Understand vulnerability injection risks
   - Recognize authentication and credential exposure
   - Know intellectual property risks
   - Understand supply chain security implications

2. **Explain code ownership issues with AI-generated code**
   - Understand licensing implications of AI-generated code
   - Know how different AI tools handle copyright
   - Articulate organizational responsibilities
   - Navigate legal gray areas

3. **Create a 1-page "AI Usage Guidelines" document for your team**
   - Define acceptable and unacceptable use cases
   - Address security and privacy requirements
   - Establish code review practices
   - Set expectations for AI assistance vs. original work

4. **Correctly identify 5/5 scenarios of appropriate vs inappropriate AI use**
   - Distinguish between helpful and harmful AI usage
   - Apply ethical reasoning to real scenarios
   - Consider security implications
   - Balance learning vs. efficiency

5. **Understand PII handling in AI prompts**
   - Recognize what constitutes PII
   - Know how to sanitize data before using AI tools
   - Understand data retention policies
   - Apply privacy-first practices

6. **Articulate when to prioritize learning over AI assistance**
   - Identify fundamental concepts that require manual practice
   - Recognize when AI hinders skill development
   - Balance productivity with learning goals
   - Make intentional choices about AI usage

## Topics Covered

- Data privacy and security concerns with AI tools
- Code ownership and licensing issues with AI-generated code
- Intellectual property considerations
- Bias in AI models and outputs
- When NOT to use AI (security-sensitive code, learning fundamentals)
- Creating organizational AI usage guidelines
- PII and sensitive data handling
- Responsible AI usage practices

## Practical Exercises

Throughout Day 2, work on:
- Reviewing code snippets for appropriate AI usage
- Analyzing case studies of AI misuse
- Creating team AI usage guidelines
- Identifying security risks in scenarios
- Practicing PII sanitization techniques
